# Weekly Macro Line Brief

## 邊界
Python 只產生並儲存週分析到 `t_market_analyses`。LINE 推送、webhook、群組觸達由 Java 系統負責。

## 目標
輸出一則能快速閱讀的每週市場摘要。重點不是列新聞，而是判斷市場這週到底在交易什麼，並推導下週台股配置方向。

## 分析窗口
- 預設看近 7 天事件。
- 若本週事件需要背景，可補充近 2-4 週脈絡，但主結論仍以本週為主。
- 預設投資視角是未來 1-4 週。

## 核心流程
每次輸出都必須照這個順序思考與呈現：

1. 總經 Regime
2. 利率與流動性
3. 景氣循環
4. 市場情緒
5. 台股配置
6. 風險與資料缺口

## 各段規則

### 1. 總經 Regime
先用 1-3 句定義本週市場主線。

可用標籤：
- 通膨黏著
- 降通膨延續
- 成長疑慮
- 軟著陸交易
- 流動性寬鬆
- 信用壓力
- AI / 科技權值主導
- 風險偏好擴散或收縮

必須回答：
- 這週市場主要在交易什麼？
- 這是基本面推動，還是估值/情緒/倉位推動？
- 主線對未來 1-4 週是否有延續性？

### 2. 利率與流動性
用條列處理數字，避免長段落。

優先檢查：
- CPI / PCE / PPI / 就業 / 薪資
- FOMC、Fed 官員、利率期貨或 Fed path
- 2Y、10Y、10Y-2Y、實質利率
- DXY、SOFR
- Fed balance sheet、RRP、TGA、reserves
- 信用利差、HY spread、金融壓力指標

必須回答：
- 利率是幫助科技股估值，還是壓抑估值？
- 流動性是在補充風險偏好，還是在抽水？
- 信用壓力有沒有升高？

### 3. 景氣循環
判斷目前更像擴張、放緩、軟著陸、衰退風險或庫存循環切換。

優先檢查：
- 消費、零售、薪資、失業率、初領失業金
- ISM / PMI、製造業與服務業
- 企業財報與 guidance
- 銀行放款、信用條件、商辦或區域銀行壓力
- 半導體、AI server、PC / 手機 / 庫存週期

必須回答：
- 成長是否足以支撐風險資產？
- 景氣變化對科技股、金融股、循環股、防禦股的影響是什麼？

### 4. 市場情緒
判斷漲跌是否有基本面支撐，還是偏倉位、軋空、追價或避險。

優先檢查：
- Nasdaq、SOX、S&P 500、Russell 2000
- VIX、信用 ETF、KRE / XLF、HYG / LQD
- 外資與期貨、台指期、選擇權 put/call
- 重大人物或帳號發言，例如 Trump、Elon Musk
- 市場是否「只漲權值，不漲大盤」

必須回答：
- 情緒偏多、偏空，還是分化？
- 是否有過熱、追價或開高走低風險？

### 5. 台股配置
把前面四段轉成台股配置，不要直接從新聞跳到個股。

可觀察方向：
- 半導體 / AI server / CoWoS / HBM / 散熱 / 電源 / PCB
- 金融股
- 高股息與防禦股
- 原物料、塑化、航運、航空
- 中小型題材股與 OTC 風險偏好

必須回答：
- 下週台股偏進攻、觀望，還是防守？
- 哪些族群值得追蹤？
- 是否適合短線交易，或只適合觀察？

### 6. 風險與資料缺口
列出最重要的反方觀點與資料缺口。

必須包含：
- 哪個數據會推翻目前判斷？
- 還缺哪些資料確認？
- 若資料不足，要降低信心，不可硬下結論。

## 格式要求
- 使用繁體中文。
- 手機可讀，短段落、短句、條列。
- 每段 1-3 bullets 或一個短段落。
- 總長度建議 700-1200 字。
- 不要輸出程式碼區塊。
- 不要編造資料。
- 必須區分「本地事件資料」、「外部驗證資料」、「推論」。

3) 景氣循環

4) 市場情緒

5) 台股配置
